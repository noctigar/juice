商品購入API¶

内容¶

  1. 指定された商品を検索
  2. 料金が足りないかを判定する
    • 料金が足りたら
      1. 精算する
      2. 売上のログデータを生成する
      3. 該当商品の在庫を-1する
    • 料金が足りなかったら
      1. エラーメッセージを返す

リクエスト¶

/item/:id/buy

params = payment

返ってくる値¶

  • 商品名
  • 購入時間
  • 投入料金-商品の値段（お釣り）

説明¶

  • ユーザーが商品購入の際の処理

補足¶

  • インプットパラメータとしてURLに「payment」が必須です。（例：~~/item/1/buy?payment=130）
  • 「payment」が入っていない、または数字ではない場合、エラーメッセージを返します。
